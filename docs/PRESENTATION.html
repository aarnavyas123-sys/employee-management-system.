01. Problem Statement

EMS Portal

Manual HR and directory management methods lead to operational lags, data leakage, and security risks:

📁 Manual Folder Tracking

Managing employees, designations, and equipment via paper lists or spreadsheets leads to error-prone directories.

⏰ Attendance Discrepancies

No automated verification of daily clock-in timestamps, shift durations, or late status flags.

💬 Approval Bottlenecks

Leave request processing delayed by lack of hierarchical routing structures and real-time email notifications.

02. Scope & Objectives

EMS Portal

EMS provides a unified enterprise panel ensuring operational transparency and data safety:

  • Role-Based Access Control (RBAC): Secure panels isolated for Admin, HR, Manager, and Employee permissions.
  • Real-time Shift Computations: Clock-in/out attendance logging with automated late arrivals threshold checks.
  • Two-Stage Leave Signature: Multi-level validation routing from employee request, manager pre-approval, to final HR signoff.
  • Asset Audit Logs: Organizational inventory assignments, tracking returns, status history, and security logs.

03. System Architecture

EMS Portal

Client Layer (Frontend)

  • Vite + React (v19) Single Page App
  • React Router DOM (v7)
  • Bootstrap 5 responsive styles
  • Axios Interceptor for headers
  • Recharts dynamic charts mapping

Server Layer (Backend)

  • Node.js + Express API Gateway
  • PostgreSQL DB Connection Pool
  • JWT validation middleware
  • bcrypt password salting & hash
  • Winston structured log levels

DevOps & Hosting

  • Vercel client SPA deployment
  • Render API server instances
  • Neon Cloud PostgreSQL Server
  • Docker Containerized stack setup
  • Cron Job approval notifications

05. Core Feature Highlights

EMS Portal

🔐 Access Control (RBAC)

Role authorization prevents unauthorized endpoints. Admin has full dashboard edits; Employees register shift punches; Manager reviews leaves; HR completes profiles.

📅 Shift Attendance Clock

Daily entry is recorded on click. Auto-calculates late status (if check-in is past 09:00 AM) and shift hours on clock-out.

📊 Security Audit Logs

Admin exclusive security module records critical events (e.g. employee creation, deletions, role updates) with operators, status changes, and timestamp logs.

06. Engineering Enhancements

EMS Portal

Best practices implemented for production-ready reliability:

🔒 Backend Guardrail Hooks

Password hashing via bcrypt. Express API requests verified using custom authMiddleware JWT signature validation checks.

📝 Winston Logger Integration

Console statements replaced with structured winston logging, mapping distinct log files (error.log, combined.log) to audit production events.

🌐 CORS Origin Validation

Configured CORS allowed origins lists dynamically matching production URLs to prevent unauthorized domain API spoofing.
