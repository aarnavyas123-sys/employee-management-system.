Enterprise Employee Management System (EMS)

A comprehensive full-stack solution built with React, Express, and PostgreSQL designed to manage enterprise human resource operations, attendance workflows, and assets.

Prepared by: Aarna Vyas

1. Project Overview & Features

General Summary

The Enterprise Employee Management System (EMS) is a production-ready Web Application designed to streamline corporate workflows, attendance tracking, hierarchical leave request signing, and organizational inventory audits.

By enforcing Role-Based Access Control (RBAC), the platform ensures users get access to exactly what their profile defines, protecting sensitive data such as salaries and audits.

🔐 Multi-Role RBAC

Delineates control sets across Admin, HR, Manager, and standard Employee roles.

📅 Attendance System

Punch-in tracking with dynamic detection of late arrivals (>09:00 AM) and shift computations.

📝 Leave Approval

Hierarchical leaves workflow (Employee -> Manager Approve -> HR Confirm -> Balance updates).

3. Database Schema Design

Relational Database Structure

Designed on PostgreSQL to enforce foreign key mappings, cascading deletes, and optimize search filters via named indices.

Table Name Description Key Columns
users Credentials, passwords, active status, roles id (PK), email (Unique), password, role, status
departments Organizational division mappings id (PK), department_name (Unique), manager_id (FK)
employee_profiles Core profile data linking to users table id (PK), user_id (FK), name, salary, remaining_leaves
leave_requests Hierarchical leave request storage id (PK), employee_id (FK), status, total_days, from_date
attendance Daily clock-in/out attendance logs id (PK), employee_id (FK), clock_in, clock_out, status
assets Company inventories and equipment catalogs id (PK), asset_name, serial_number (Unique), status
audit_logs Security logs tracking crucial adjustments id (PK), user_id (FK), action, details, created_at

Optimization Indexes Implemented

-- High search optimization indexes
CREATE INDEX idx_employee_name ON employee_profiles(name);
CREATE INDEX idx_employee_department ON employee_profiles(department_id);
CREATE INDEX idx_department_name ON departments(department_name);

4. Core Module Setup

1. Authentication & Security Guardrails

Passcodes are salted using bcrypt and transmitted securely. Requests carry custom JSON Web Tokens (JWT) verified before route handling.

2. Attendance tracking

An automated clock-in tracker computes check-in hours. Incurs a "Late" status if punched after 09:00 AM.

3. Leave Request Workflow

A multi-stage signature workflow allowing employee request, manager pre-approval, and final HR sign-off. The leave balance is adjusted upon HR approval.

5. Core API References

Public & Protected Routes Mapping (Prefix: /api/v1)

Method Endpoint Path Access Role Action Description
POST /auth/login Public Verify login and return JWT token
POST /auth/signup Public Register a basic employee user profile

POST /attendance/clock-in Employee Punch check-in and calculate late flag
PUT /leaves/approve/:id Manager Manager pre-approval level check
PUT /leaves/hr-approve/:id HR Confirm HR signoff and update leave days balance
GET /audit-logs Admin Only Admin compliance audit details search
GET /health Public API Health check test route (UP/DOWN)
